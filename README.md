# まなびしりとり

ブラウザだけで遊べる、教育型の日本語しりとりゲームです。フェーズ1では「日本の都道府県」テーマで、ひとり練習、コンピュータ対戦、みんなで対戦を遊べます。

## 起動方法

ビルドやサーバーは不要です。

1. このリポジトリをダウンロードまたはクローンします。
2. `index.html` をブラウザで開きます。
3. ローカル配信で確認する場合は `python3 -m http.server 4173` を実行し、`http://127.0.0.1:4173/` を開きます。

## GitHub Pagesでの公開方法

1. GitHubのリポジトリ設定で Pages を開きます。
2. 公開元をこのリポジトリのブランチとルートディレクトリにします。
3. 保存後、表示されたURLを開きます。ビルド手順は不要です。

## 遊び方

1. タイトル画面で「ゲームをはじめる」を押します。
2. モードを選びます。
   - ひとりで練習: 制限時間なしで練習できます。
   - コンピュータと対戦: プレイヤーとCPUが交互に答えます。
   - みんなで対戦: 2〜4人で1台の端末を交代して遊びます。
3. 設定画面で制限時間や詳細設定を確認し、「この設定ではじめる」を押します。
4. 最初のプレイヤーは任意の都道府県を回答できます。2人目以降は前の言葉の最後の文字から始まる都道府県を入力します。

## 都道府県名の判定ルール

- 表示名は正式な都道府県名です。
- 入力は正式名、省略名、ひらがな、カタカナ、半角カタカナ、前後の空白に対応します。
- 初期設定では、しりとり判定に使う読みから「都・道・府・県」を除きます。
  - 例: `東京都` / `東京` / `とうきょうと` は同じ `tokyo` として扱い、末尾は `う` です。
  - 例: `京都府` / `京都` は同じ `kyoto` として扱い、末尾は `と` です。
- 詳細設定の「都・道・府・県を含めて判定」をONにすると、行政区分を含む読みで末尾判定します。
  - 例: `東京都` の末尾は `と`、`京都府` の末尾は `ふ` になります。
- 使用済み判定はid単位です。`東京` を使った後に `東京都` や `とうきょうと` は使えません。
- 登録されていない未知の漢字は推測せず、登録済みの `name` / `reading` / `aliases` と照合します。

## 小書き文字の正規化

- 標準照合では `normalizeForLookup()` を使い、小書き文字を保持します。
  - `キョウト` と `ｷｮｳﾄ` は `きょうと` として扱います。
  - `きょうと` と `きようと` は標準照合では区別します。
- キッズ向けなどの緩い比較では `normalizeForLooseComparison()` を使い、必要な場合だけ小書き文字の大文字化や濁点許容を行います。
- 標準の `reading` データは変換せず、東京都の読みは `とうきょうと` のまま保持します。


## テーマ共通化の方針

- テーマ定義は `core.js` の `THEME_DEFINITIONS` に集約しています。
- ゲームで使う言葉は `getThemeWords(window.MANABI_DATA, themeId)` で取得します。
- テーマごとの件数とプレイ可否は `getThemeStats()` で判定します。
- 現在プレイ可能なのは `prefectures` のみです。`countries` / `animals` / `foods` / `mix` はフェーズ2以降の準備中として表示し、ゲーム開始時には選択できないようにしています。
- データ検証は共通の `validateThemeData()` を使い、都道府県だけは県庁所在地や地方区分も必須にする `validatePrefectureData()` を使います。

## みんなで対戦の脱落ルール

- 対戦人数は2〜4人です。
- 全員アクティブ状態で開始します。
- 正解すると、次のアクティブなプレイヤーへ交代します。
- 不正解、時間切れ、「ん」で終わる回答、使用済み回答、テーマ外の言葉、必要文字とつながらない回答、ギブアップで現在プレイヤーは脱落します。
- 脱落したプレイヤーには以後ターンが回りません。
- アクティブなプレイヤーが1人になった時点で終了し、最後に残ったプレイヤーが優勝です。
- 脱落順を記録し、順位は「勝者 → 最後に脱落した人 → 先に脱落した人」の順で表示します。
- 47都道府県をすべて使い切った場合、または次に必要な文字から始まる未使用候補がなくなった場合は、最後に正解したプレイヤーを勝者として終了します。

## 保存内容

`localStorage` の `manabiShiritori.phase1` に、モード別の最高スコアと覚えた言葉のidを保存します。個人情報は保存しません。localStorageが利用できない場合や壊れたJSONが入っている場合でも、ゲーム進行を優先して初期値で動作します。
