# PROJECT RULES

## 目的

このファイルは、国旗しりとりゲームの仕様破壊、デグレ、仮素材の混入、未確認変更の本番反映を防ぐための最優先ルールである。

Codex、AI、人間の作業者は、作業開始前に必ずこのファイルを読むこと。

## 作業ルール

- mainブランチへ直接コミットしない。
- 必ず新しいブランチで作業する。
- 未確認の変更をGitHub Pages本番へ反映しない。
- 仕様変更時は、先にdocsを更新してから実装する。
- UI、ゲームロジック、国データ、国旗素材を一度に大規模変更しない。
- 指示されていないリファクタリングをしない。
- 既存機能を整理、統合、削除しない。
- 変更範囲外のファイルへ触れない。
- プレースホルダーや仮データを完成品として扱わない。

## データルール

- `data/countries.js`を唯一の国マスタとする。
- 国データを複数ファイルへ重複定義しない。
- 196か国の一覧と難易度分類を勝手に変更しない。
- 国旗は正確なローカルSVGを使用する。
- 仮国旗、国コードだけの旗、ランダム配色の旗を禁止する。
- 実行時に外部国旗URLへ依存しない。
- ヒントに国名を直接含めない。
- うんちくに複数国共通の空疎な文章を流用しない。

## ゲーム仕様の固定事項

- 1人モードはライフ3個。
- 3回間違えると終了する。
- 10問固定ではなくエンドレス形式とする。
- スコアは累計正解数とする。
- 連続正解数を記録する。
- 自己ベストを`localStorage`へ保存する。
- 行き止まり時は使用済み国を段階的に再利用する。
- 難易度は初級、中級、上級、激ムズの4段階とする。
- 難易度は累積式とする。
- CPU対戦の出題難易度とCPU強さは別設定とする。
- ヒントは段階式とする。
- 回答後に学習情報を表示する。
- 回答後の表示時間は設定可能とする。

## UIルール

- スマホ縦画面を最優先する。
- 3択は常に横並びにする。
- 国旗カードに大きな縦余白を作らない。
- 国旗の比率を崩さない。
- ボタンを必要以上に巨大化しない。
- 画面の優先順位は次のとおりとする。
  1. 次の文字
  2. 選択肢
  3. 現在の国
  4. ステータス
  5. 補助機能

## 明示的な指示なしに変更禁止

- 収録196か国の構成
- 難易度分類
- 正確な国旗SVG
- 3ミス制
- エンドレス形式
- ヒントに国名を直接出さない方針
- `data/countries.js`を唯一の国マスタにする構成
- オンライン対戦を後回しにする方針

**「未確認の仮実装、仮素材、プレースホルダー、共通文データを完了とみなしてはいけない。」**
