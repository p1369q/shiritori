# COUNTRY DATA POLICY

## 唯一の国マスタ

- `data/countries.js`を唯一の国マスタとする。
- 別ファイルへ国データを重複させない。

## 必須項目

各国に最低限、以下を持たせる。

- `id`
- `code`
- `name`
- `reading`
- `difficulty`
- `region`
- `flagFile`
- `capital`
- `populationLabel`
- `populationYear`
- `areaLabel`
- `languages`
- `currency`
- `trivia`
- `hints`

さらに、以下のうち最低1つ以上を持たせる。

- `famousFoods`
- `famousPlaces`
- `specialties`

## 禁止事項

- 同じ`trivia`を複数国へコピーすること。
- 「世界の国旗しりとりで登場する国です」のような共通文を使うこと。
- 人口の基準年を省略すること。
- 架空の人口、面積、首都を記載すること。
- 国名を含むヒントを作ること。
- 国旗の仮画像を使うこと。
- 空欄を適当な文章で埋めること。

## 人口

- 「約○万人」の形式とする。
- `populationYear`を必須とする。
- 厳密な1人単位にしない。
- 出典や基準年が異なる場合は見やすく丸める。

## 子ども向け文章

- 小学生にも理解できる文章にする。
- 30〜70文字程度にする。
- 難しい制度名だけで説明しない。
- 簡略化しても事実を変えない。

## 難易度

- 指定済みの4段階分類を厳守する。
- 自動分類しない。
- 人口、面積だけで難易度を決めない。
- 上位難易度は下位難易度を含む累積式とする。

## データ変更時

- 変更理由を書く。
- 出典を確認する。
- 変更国だけでなく関連テストを実行する。
- 件数が196から変わっていないことを確認する。
